# 零阶段人工协作流程

> 状态：当前基线
> 所属：协作治理
> 规则效力：零阶段人工协作规则
> 维护角色：产品负责人


## 目的

零阶段是 Clowder AI 还没有自己的聊天室和自动 Harness 之前的阶段。在这个阶段，由用户人工协调 Agent 并记录过程。

这不是最终工作模式，而是对未来 A2A、Harness、Review 和门禁机制的人工模拟。

## 零阶段原则

用户临时承担 Harness 职责：

- 分发上下文。
- 分配角色。
- 路由任务。
- 收集产出。
- 强制 Review。
- 跟踪决策。
- 升级冲突。
- 防止方案确认前进入开发。

## 推荐零阶段流程

1. 产品负责人完成产品文档包。
2. 用户将产品文档包交给 Codex。
3. Codex 以系统架构师身份工作。
4. Codex 只输出架构设计。
5. 产品负责人审查架构是否覆盖产品需求。
6. 需求覆盖性审查通过后，再拆分实现任务。
7. 用户手动打开相关 Agent 客户端。
8. 用户分别给每个 Agent 提供产品上下文、A2A 协议、对应角色定义、架构文档、具体任务、Review 与升级规则。
9. Agent 尽量在独立上下文或工作区中产出结果。
10. 用户收集产出并触发交叉 Review。
11. 用户执行质量门禁和交付规则。
12. 用户记录复盘。

## 零阶段输入要求

在聊天室和自动 Harness 建成前，每次给 Agent 分配任务时，用户需要明确：

- 当前 Agent 的身份和角色边界。
- 需要阅读的产品文档和任务文档。
- 当前任务目标。
- 当前任务不得越过的边界。
- 需要由谁 Review。
- 哪些情况必须升级给用户。

## 零阶段 Review 要求

人工触发 Review 时，应要求 Review 方检查：

- 是否符合需求。
- 方案是否正确。
- 是否存在角色相关问题。
- 风险和边界场景。
- 测试或验证是否充分。
- 是否需要用户确认。

Review 结论必须是以下之一：

- 通过。
- 需要修改。
- 存在争议。
- 需要用户确认。

## Agent CLI 不可用时的降级处理

在自动 Harness 建成前，若某个 Agent CLI 不可用，可以继续采用人工 Agent 接力方式推进零阶段，但必须满足：

- 明确记录不可用的 Agent 和失败原因。
- 明确说明这是临时降级，不代表自动化三 Agent 闭环已经具备。
- 不得用模拟结果静默替代真实 Agent 输出。
- 相关工作项必须标记为阻塞或降级执行。
- 恢复自动化闭环前，必须重新验证 Agent CLI 可用性。

## 零阶段成功标准

零阶段在以下条件满足时算成功：

- 基于产品文档产出系统架构设计。
- 产品负责人确认架构覆盖产品需求。
- 首批实现任务带着角色上下文被分配。
- Agent 产出经过非作者 Review。
- 手工流程暴露的问题被记录为未来 Harness 需求。
