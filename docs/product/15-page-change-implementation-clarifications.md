> 所属：产品

> 维护角色：产品负责人
> 日期：2026-06-19

## 背景

`P0-14`、`P0-15`、`P0-16` 已批准后，T2 和 T3 执行 Agent 已分别输出变更影响评估：

- [../execution/25-change-impact-assessment-p0-14-16.md](../execution/25-change-impact-assessment-p0-14-16.md)
- [../execution/26-t2-change-impact-assessment-p0-14-16-by-codex.md](../execution/26-t2-change-impact-assessment-p0-14-16-by-codex.md)

两份评估中存在部分待确认问题。本文件给出产品侧澄清。架构实现方式仍由系统架构师和执行计划决定，但不得违背本文中的产品边界和验收口径。

## 1. T2 是否必须回补页面录入

结论：必须回补。

T2 不能只保留 CLI，然后把页面录入全部推迟到 T13。原因是 `P0-14` 已批准，页面是首版主入口；T2 当前任务已被执行计划重定为“页面工作项录入与类型选择/识别最小入口”。

T2 的边界是：

- 做最小页面录入入口。
- 用户可通过页面提交功能需求或 Bug 修复。
- 页面入口必须真实创建 WorkItem。
- 创建后展示最小初始状态：ID、类型、当前状态、标题或目标、低置信度或待确认提示。
- CLI 保留为零阶段或内部入口，但不能作为 T2 新基线验收入口。

T13 的边界是：

- 完整页面级用户参与主界面。
- 包括统一聊天室与时间线、详情、阻塞确认、Review、门禁和复盘入口。
- 不把 T13 的完整范围塞进 T2。

## 2. T3 是否必须改代码

产品侧不要求 T3 必须改代码，但要求 T3 证明能支撑页面读取。

如果当前 Store 和 `list(filter)` 已能支撑页面所需查询，可以不改 T3 代码。但 T3 必须在交付说明或验证中明确：

- 工作项详情如何读取。
- 时间线如何读取。
- 阻塞和待确认项如何读取。
- Review、门禁、复盘摘要如何读取。
- 最近一次关键结论如何得到。

如果这些能力由页面层、应用服务层或查询投影聚合，也可以接受；具体落点由架构决定。

## 3. T2/T3 持久化出口

结论：必须统一，且应在 T5 状态机启动前完成。

产品不指定具体实现方式，但不接受两个 WorkItem 来源长期并存：

- `data/work-items/<id>.json`
- `data/work-items.json`

最终必须形成一个事实来源。T2 页面入口和 CLI 内部入口都应走统一 WorkItem 创建和持久化出口。

## 4. 最近一次关键结论

首版定义：

最近一次会影响用户理解当前状态、下一步动作或风险判断的结构化结论。

可来自：

- A2AEvent 的 `conclusion` 或 `next_action`。
- ReviewRecord 的 `result`、`findings` 或 `required_fixes`。
- QualityGateRun 的 `result`、`failure_reason` 或 `final_status`。
- EscalationRecord 的 `recommended_next_step` 或 `user_decision`。
- RetrospectiveMemory 的复盘结论。

如果当前工作项还没有关键结论，页面可以显示“待形成”或等价状态，不要求伪造结论。

## 5. T13 页面范围

产品不要求首版拆成多个页面，也不要求高保真设计、完整设计系统或复杂路由。

首版可以是一个本地页面或单页多区域界面，但必须覆盖五类能力：

- 工作项录入入口。
- 统一聊天室与时间线。
- 工作项详情视图。
- 阻塞与人工确认视图。
- Review、门禁与复盘视图。

页面数量、路由和组件结构由架构和实现决定。

## 6. MiniMax 参与节奏

由于 MiniMax 客户端当前操作不友好，允许在 T2 最小页面录入入口搭建完成后，再引入 MiniMax。

这是零阶段执行上的临时调整，不代表取消 MiniMax 的产品职责。

执行口径：

- T2 最小页面录入可以先由 Codex / Claude 完成和 Review。
- T2 页面只按“功能骨架”验收，不按完整交互体验验收。
- MiniMax 必须在 T13 页面主界面或 T16 E2E 前参与页面体验 Review。
- 如果 MiniMax 后续发现影响用户理解、确认或操作效率的问题，应作为 T13 或 T16 修复项处理。

## 产品裁决摘要

- 采纳 T2 评估中“当前 T2 不满足新页面主入口基线，必须补齐最小页面录入入口”的判断。
- 不采纳 T3 评估中“页面录入可以全部放到 T13，T2 仅追加文档说明”的建议。
- 接受 T3 “不必强制改代码”的判断，但前提是 T3 能证明页面查询视角成立。
- T2/T3 持久化出口统一是进入 T5 前的必要条件。
